# Improved Part-Color Image Import (LDraw from Rebrickable)

## Goal

Replace the current CSV-based (`inventory_parts.csv` `img_url`) part-color image
import with one that downloads Rebrickable **LDraw** rendered images, packaged as
per-color ZIP archives:

```
https://cdn.rebrickable.com/media/downloads/ldraw/parts_{color.rebrickable_id}.zip
```

Each ZIP contains PNG files named after the part:

```
{part.rebrickable_id}.png        // stem == parts.rebrickable_id
{part.rebrickable_id}-1.png      // stem (incl. trailing -N) == parts.rebrickable_id
```

The filename stem (filename minus `.png`) **is** the part's `rebrickable_id`
(Rebrickable part numbers frequently contain dashes, e.g. `92593-1`). We match the
full stem directly against `parts.rebrickable_id`; no splitting on `-`.

Images are attached to the `part_colors` pivot in a **new dedicated `ldraw`
media collection** (separate from the existing `image` collection) so future
image sources can coexist.

## Decisions (confirmed with user)

- **Skip mechanism:** nullable `ldraw_images_imported_at` timestamp on `colors`.
  A color's ZIP is skipped if it was imported within a freshness window, unless
  `--full` is passed. Freshness window is a job constant (default 7 days).
- **Command:** keep `part:import-color-images`, add `--full` flag to force
  re-download of every color regardless of freshness.
- **Job granularity:** one queued job per color (`ImportColorLdrawImagesJob`),
  dispatched by a dispatcher job/command. Keeps memory bounded and parallel.
- **Collection:** new `PartColor::LDRAW_IMAGE_COLLECTION = 'ldraw'` (singleFile),
  with the same `preview` / `large` conversions.
- **Logging:** each per-color job logs its own counts; a final summary logs total
  `part_colors` having an ldraw image (via `Log` facade).
- **Filename mapping:** stem == `parts.rebrickable_id` (strip only `.png`).

## Implementation Steps

### 1. Migration: track LDraw import freshness on `colors`

New migration `add_ldraw_images_imported_at_to_colors_table`:

```php
Schema::table('colors', function (Blueprint $table) {
    $table->timestamp('ldraw_images_imported_at')->nullable()->after('is_transparent');
});
```

- Add `ldraw_images_imported_at` to `Color::$fillable` and cast to `datetime` in
  `Color::casts()`.

### 2. `PartColor` — add the `ldraw` media collection

In `app/Models/Pivots/PartColor.php`:

- Add `public const string LDRAW_IMAGE_COLLECTION = 'ldraw';`
- In `registerMediaCollections()`, also register the `ldraw` collection as
  `singleFile()`.
- In `registerMediaConversions()`, register `preview` (150x150 Contain) and
  `large` (600x600 Contain) — these conversions already apply across collections;
  keep as-is so they generate for `ldraw` media too.

### 3. New download capability for LDraw ZIPs (binary entries)

The existing `RebrickableDownloadService` only yields CSV rows. Add a method to
stream **named binary entries** from a ZIP without loading everything into memory.

Extend the `RebrickableDownloader` contract:

```php
/**
 * Yield each file entry of a remote ZIP as [name => string contents].
 * Returns void/empty generator if the archive is missing (404) or invalid.
 *
 * @return Generator<string, string>  filename => raw bytes
 */
public function retrieveZipEntriesFromUrl(string $url): Generator;
```

Implement in `RebrickableDownloadService::retrieveZipEntriesFromUrl()`:

- `Http::get($url)`; if `$response->failed()` (e.g. 404 for a color with no LDraw
  parts), return an empty generator (no error thrown — a missing ZIP is expected
  for some colors).
- Write body to a `tmpfile()`, open with `ZipArchive`.
- Iterate `0..$zip->numFiles-1`; for each, get name via `getNameIndex($i)`, skip
  directory entries, read contents via `getFromIndex($i)` (or stream via
  `getStream` + `stream_get_contents` to bound peak memory), `yield $name => $bytes`.
- Close stream/zip, clean tmpfile.

Notes:
- A single color ZIP can hold thousands of PNGs; yielding one entry at a time keeps
  memory bounded. Each PNG is small (rendered thumbnail), so reading per-entry is fine.
- Keep the existing CSV method untouched.

### 4. New per-color job: `ImportColorLdrawImagesJob`

`app/Domain/Part/Jobs/ImportColorLdrawImagesJob.php` (replaces the old
`ImportPartColorImagesJob`; old file removed).

Constructor: `public function __construct(public int $colorId) {}`

`handle()`:

1. Load the `Color` (`find($colorId)`); bail if missing.
2. Build URL: `https://cdn.rebrickable.com/media/downloads/ldraw/parts_{$color->rebrickable_id}.zip`.
   (Base URL in `config/rebrickable.php`, e.g. `ldraw_parts_zip_url` template, to
   match how other Rebrickable URLs are handled — or inline constant if config has
   none; see Open Question.)
3. Preload `parts.rebrickable_id => id` map once
   (`Part::query()->pluck('id', 'rebrickable_id')`).
4. Stream entries via `retrieveZipEntriesFromUrl($url)`. For each entry:
   - Skip if not `.png`.
   - Stem = filename without `.png` and without any directory prefix.
   - Look up `partId` from the map; skip if unknown.
   - `firstOrCreate` the `PartColor` for `(part_id, color_id)`.
   - If it already has media in `LDRAW_IMAGE_COLLECTION`, skip (idempotent re-run).
   - `addMediaFromString($bytes)->usingFileName("{$part->rebrickable_id}-{$color->rebrickable_id}.png")
     ->toMediaCollection(PartColor::LDRAW_IMAGE_COLLECTION)`.
   - Increment a local `$attached` counter.
5. Set `$color->ldraw_images_imported_at = now(); $color->save();`.
6. `Log::info` a per-color summary: color id/rebrickable_id, entries seen,
   matched parts, images attached, unknown-part skips.

Memory/perf:
- One ZIP per job; entries streamed. No `Http::pool` needed since bytes already
  come from the ZIP (single HTTP download per color instead of per-image).
- This is the big efficiency win vs. the old per-image CDN fetches.

### 5. Dispatcher: command + a dispatcher job

Repurpose `ImportPartColorImagesCommand` (`part:import-color-images`):

```php
protected $signature = 'part:import-color-images {--full}';
```

`handle()`:
- Query colors to process:
  - Without `--full`: colors where `ldraw_images_imported_at IS NULL` OR older than
    the freshness window (`now()->subDays(FRESHNESS_DAYS)`).
  - With `--full`: all colors.
- For each, `ImportColorLdrawImagesJob::dispatch($color->id)`.
- Optionally dispatch a final `LogLdrawImportSummaryJob` (chained after, or a
  simple `Log::info` of how many colors were queued). For an accurate end-of-run
  count of part_colors-with-image we add a small summary job (Step 6) dispatched
  via a `Bus::batch(...)->then(...)` or `->finally()` so it runs after all
  per-color jobs finish.

Freshness window:
- Constant on the dispatcher/job, e.g. `protected int $freshnessDays = 7;`
  (user did not opt into config; keep as a documented constant, easy to tune).

### 6. End-of-run logging summary

Per the requirement "at the end we want to know how many part-color combinations
have an image":

- Use `Bus::batch($jobs)->finally(function () { ... })` in the command so that after
  all per-color jobs complete, we log:
  - `PartColor::whereHas('media', fn ($q) => $q->where('collection_name', PartColor::LDRAW_IMAGE_COLLECTION))->count()`
  - i.e. total part-color combinations that now have an LDraw image.
- Implemented as a small `LogLdrawImportSummaryJob` (or closure in the batch
  `finally`) using `Log::info`.
- Note: requires `jobs`/`job_batches` tables. `job_batches` migration may need to be
  published if absent (check `database/migrations`; `0001_01_01_000002_create_jobs_table.php`
  exists — confirm it includes `job_batches`; if not, add it). See Open Question.

### 7. Remove / replace old CSV image logic

- Delete `app/Domain/Part/Jobs/ImportPartColorImagesJob.php` (CSV-based).
- The old `PartColor::IMAGE_COLLECTION` ('image') stays defined for backward
  compatibility / future non-LDraw sources, but is no longer written by this flow.

## Resolved Decisions

1. **`job_batches` table / summary** — CONFIRMED: use `Bus::batch(...)->finally(...)`.
   Add a `job_batches` migration if the existing jobs migration lacks the table.
   The accurate end-of-run total is logged in the batch `finally()` callback.
2. **404 / missing color ZIP** — CONFIRMED: treat as "no images for this color";
   no error thrown, and `ldraw_images_imported_at` IS still stamped so we don't
   re-download every run until the freshness window passes.
3. **LDraw base URL** — DEFAULT (not explicitly answered): add the ZIP URL template
   to `config/rebrickable.php`, consistent with other Rebrickable settings.
4. **Freshness window** — DEFAULT (not explicitly answered): **7 days** as a
   documented constant on the command/job (not config).
